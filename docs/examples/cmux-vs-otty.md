# Example: cmux vs Otty

This bundled report demonstrates how FitLens treats products with different
levels of public evidence.

## Evidence boundary

### cmux

- Product site: <https://cmux.com/>
- Public repository: <https://github.com/manaflow-ai/cmux>
- Open-source claims may be checked against repository metadata, the license,
  and README.

### Otty

- Product site: <https://otty.sh/>
- Documentation: <https://docs.otty.sh/>
- The report labels feature statements as vendor claims because the official
  site does not currently link to a public source repository.
- “No public repository was found” is an observation, not proof that no source
  exists.

## Why the recommendation is personal

The bundled profile values openness, coding-agent workflows, native
performance, and automation. With those weights, cmux wins because its public
implementation and programmable interfaces matter more than Otty’s additional
productized workflow features.

Changing the weights can change the winner. That is the point: FitLens scores
fit for a stated workflow, not universal product quality.
